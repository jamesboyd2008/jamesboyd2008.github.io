Enumerable#group_by

The Enumerable module is a collection of methods. Most of them seem to have to do with arrays. Here is the documentation. The first paragraph at the top does a better job than me at explaining Enumerable. Enumerable has a lot of methods for sorting out arrays in various fashions. ".group_by" is the method of focus for this blog post. This method is called by an array, range, or any class that has a ".each" method. It sorts the collection into groups, and returns those groups as key/value pairs in a hash. That's it, in short. It runs each item in the collection through a block, and based on the outcome of that block, sorts each item into it's appropriate key/value pair. Here's two examples:

(1..10).group_by { |i| i.even? }
# => {false=>[1, 3, 5, 7, 9], true=>[2, 4, 6, 8, 10]}

In the above example, group_by goes through each item in the range, and determines whether it is even (because that's what the code block does). Since every item in the range, when evaluated by the block, is true or false, those are the only two keys in group_by's result hash.

grab_bag = ["hammer", 12, :yipee, 7, "Dick Van Dyke", :skippy]
grab_bag.group_by { |elem| elem.class }
# => {String=>["hammer", "Dick Van Dyke"], Fixnum=>[12, 7], Symbol=>[:yipee, :skippy]}

In the above example, group_by goes through each item in the array, runs it through the code block which spits out its class, and returns a hash with 3 keys, because there are only 3 different classes represented in grab_bag. The values for each key are the elements from grab_bag with a class type of the corresponding key.
