Ruby Hashes vs. JavaScript Objects

January 25, 2016

Ruby hashes are similar to JavaScript objects. Ruby hashes are composed of key/value pairs, and the hash itself usuall has a name. JavaScript objects are a variable that contain names pointing to values. Those values can be numbers, arrays, strings, functions, other objects, or what have you. I don't recall ever defining a Ruby method within a hash, but in JavaScript, you can define functions inside an object. JavaScript constructor functions are more comparable to Ruby classes. They aren't quite the same, though. Ruby classes tend to have more default qualities, I think.
